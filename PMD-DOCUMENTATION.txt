1. OVERVIEW
================================================================================

Project Minimalist Design (PMD) is a design system built on principles of 
simplicity and practicality. The system emphasizes YAGNI (You Aren't Gonna 
Need It) and DRY (Don't Repeat Yourself) principles.

Key Characteristics:
- Variable-based architecture for rapid theme iteration
- Lightness-based hierarchy over decorative elements
- OKLCH colorspace for perceptual uniformity
- rem-based scaling (base-16)
- Framework-agnostic implementation

Light Scheme OKLCH Mapping:
- 100x = oklch(0 0 0)       - Black/lowest priority
- 88x  = oklch(0.28 0.032 0) - Primary lightness
- 80x  = oklch(0.2 0.032 0)  - Secondary lightness
- 72x  = oklch(0.32 0.052 0) - Accent/Subtext lightness
- 8x   = oklch(0.88 0.056 0) - Base lightness
- 4x   = oklch(0.92 0.044 0) - Deep base/Background
- 0x   = oklch(1 0 0)       - White/highest priority (Tinting source)

Dark Scheme OKLCH Mapping:
- 100x = oklch(1 0 0)       - White/highest priority
- 88x  = oklch(0.88 0.056 0) - Primary lightness
- 80x  = oklch(0.8 0.1 0)    - Secondary lightness
- 72x  = oklch(0.72 0.122 0) - Accent/Subtext lightness
- 8x   = oklch(0.2 0.032 0)  - Base lightness
- 4x   = oklch(0.16 0.022 0) - Deep base/Background
- 0x   = oklch(0 0 0)       - Black/lowest priority (Tinting source)

The system serves as flexible guidelines rather than rigid specifications,
encouraging consistent visual hierarchy across diverse UI/UX projects.

================================================================================
2. USAGE GUIDELINES
================================================================================

PMD functions as a flexible guideline rather than rigid specification 
framework. Designers should adapt values contextually while maintaining 
hierarchical relationships.

Core Principles:
1. Use lightness, opacity, weight, spacing, and radius for hierarchy
2. Avoid gradients and shadows unless hierarchical clarity demands them
3. Leverage mutable variables for global adjustments
4. Maintain top-down (Z-layer) and Left-to-Right hierarchy
5. Verify both light and dark schemes for accessibility

General Implementation:
- Import palette definitions from src/palette/pmd/
- Apply font configurations from src/fonts/
- Reference variables in your framework of choice
- Test across multiple viewport sizes

================================================================================
3. HIERARCHY THROUGH VISIBILITY
================================================================================

PMD establishes hierarchy through visibility properties rather than 
decorative elements:

Hierarchy Framework (Top-Down / Z-Layer / Y-Axis):
Lightness: Primary differentiator (100x > 88x > 80x > 64x > 8x > 4x > 0x)
Opacity: Subtle layering and emphasis (100% > 80% > 64% > 40% > 24% > 12% > 8%)
Weight: Typographic emphasis (Bold > Semibold > Medium)
Spacing: Breathing room and grouping (larger = more important)
Radius: Element prominence (1rem default for all frames)

Component Hierarchy Examples:
88x - Selection/hover borders, Icon/hero elements (Highest priority)
80x - Text/label elements (Secondary priority)
88x - Dropdown/trail elements (Navigation priority)

Note: Icons take visual priority due to their symbolic nature and immediate 
recognition at glance.

================================================================================
4. FRAME CONVENTIONS
================================================================================

Standard frame specifications for all schemes:

Border: 0.125rem (2px) constant width
Radius: 1rem (16px) constant radius
Spacing: 0.5rem (8px) or 0.25rem (4px) internal padding

State Colors:
- Boost: 88x+12 (complementary highlight)
- Active: 88x (enabled primary state)
- Selection/Hover Border: 88x
- Occupied: 72x (interactive/focused elements)
- Disabled: 88x at 24% opacity
- Surface: 8x + 80x at 12% opacity (composited stack)
- Base: 4x (solid background)

For grouped components, use asymmetric radius (0.5rem on relative sides)
to create visual connection between adjacent elements.

================================================================================
5. VARIABLE SYSTEM
================================================================================

PMD uses mutable variables formatted as <lightness%>x<opacity%><+hue-offset>

OKLCH Variable Definitions (Neutral Tone L/C Values):
Variable | Dark L / C   | Light L / C  | Purpose
---------|--------------|--------------|-------------------------
100x     | 1.00 / 0.000 | 0.00 / 0.000 | Max Contrast Text/Symbol
88x      | 0.88 / 0.056 | 0.28 / 0.032 | Headers, Icons, Selection
80x      | 0.80 / 0.100 | 0.20 / 0.032 | Body Text, Description
72x      | 0.72 / 0.122 | 0.32 / 0.052 | Accent, Subtext, Links
8x       | 0.20 / 0.032 | 0.88 / 0.056 | Base Containers, Surfaces
4x       | 0.16 / 0.022 | 0.92 / 0.044 | True Background (base00)
0x       | 0.00 / 0.000 | 1.00 / 0.000 | Black/White Tinting Anchor

Each variable supports multiple opacity variants. 
Auxiliary hues use +180° offset by default: aux_hue = (primary_hue + 180) mod 360

================================================================================
6. TYPOGRAPHY SYSTEM
================================================================================

Font Hierarchy:
Bold (Fredoka Bold, Rounded Mplus 1c Bold)
- Scales: 3rem, 2.5rem, 1rem
- Usage: Major headers, logo hero elements

Semibold (Fredoka SemiBold)
- Scales: 1rem, 0.75rem, 0.625rem
- Usage: Section headers, active toggle labels

Medium (Fredoka Medium, Rounded Mplus 1c Medium)
- Scales: 1.25rem, 1rem, 0.75rem
- Usage: Body text, paragraphs, definitions

Mono (FiraCode Nerd Font)
- Scales: 1rem, 0.75rem, 0.625rem, 0.5rem
- Usage: Code snippets, HEX/OKLCH values, icons

Typographic Shade Hierarchy:
88x - Headers, titles, sections (Primary emphasis)
80x - Descriptions, body text, definitions (Secondary emphasis)
72x - Subtext, links, smaller text (Subtle emphasis)

Icon Sizing:
- Material Symbols Standard: 1.5rem (24px)
- Small: 1rem (16px)

7. PALETTE SYSTEM
================================================================================

PMD palettes are defined in YAML format mapping to base16 slots:

Base16 Slot | PMD Mapping           | Description
------------|-----------------------|--------------------------
base00      | 4x                    | True Background
base01      | 8x                    | Base Container
base02      | 8x + 80x @ 12%        | Composited Surface/Selection
base03      | 80x @ 80%             | Muted / Comments (Baked)
base04      | 72x                   | Dark Foreground / Subtext
base05      | 80x                   | Body Text / Labels
base06      | 88x                   | Headers / Selection Active
base07      | 100x                  | Highest Contrast text
base08-base0F Accents               | 72x/80x Chromatic Variants

================================================================================
8. HUE REFERENCE
================================================================================

OKLCH Hue Values (shifted ~30° from HSL):
- Red: 30°
- Orange: 60°
- Yellow: 90°
- Green: 140°
- Teal/Cyan: 195°
- Blue: 260°
- Purple: 290°
- Rose: 345°

Calculation: (primary_hue + 180°) mod 360° for Auxiliary Hue.

================================================================================
9. LIGHT SCHEME INVERSION
================================================================================

Light scheme is calculated through optimized OKLCH substitution rather than 
simple mathematical inversion to ensure perceptual legibility:

Inversion Logic:
- Background: From 4x (Dark) to 4x (Light) [0.16 -> 0.92]
- Foreground: From 88x (Dark) to 88x (Light) [0.88 -> 0.28]
- Accents: Hue and Chroma persist, Lightness adjusts to 72x level (0.32).

Opacity percentages remain unchanged. Chroma is stripped for monochrome modes.

================================================================================
10. OPACITY SYSTEM
================================================================================

Standard Opacity Values:
100% - Full visibility
80%  - Strong presence (base03 mapping)
64%  - Medium translucency
40%  - Subtle base layers
24%  - Disabled states, outlines
12%  - Surface layers (base02 alpha component)
8%   - Background surfaces
0%   - Fully transparent

================================================================================
11. SCALE SYSTEM
================================================================================

All measurements use rem units (base-16: 16px = 1rem)

Common Scales:
- 0.125rem (2px) - Standard border width
- 0.25rem (4px)  - Micro spacing
- 0.5rem (8px)   - Standard spacing
- 1rem (16px)    - Base unit / Frame Radius
- 1.5rem (24px)  - Header spacing / Icon size
- 2rem (32px)    - Prominent sections

================================================================================
12. SPACING SYSTEM
================================================================================

Internal Padding:
- 0.25rem - Compact (buttons)
- 0.5rem  - Standard (cards)
- 1rem    - Generous (containers)

External Margins:
- 0.5rem  - Tight grouping
- 1rem    - Standard separation
- 2rem    - Section breaks

================================================================================
13. EFFECTS SYSTEM
================================================================================

Backdrop Blur:
- Standard: 24% opacity
- Usage: Translucent surfaces at 8x 40%

Shadows/Gradients:
- Avoid entirely per minimalist guidelines.
- Depth is communicated via Lightness and Borders only.

14. NIX INTEGRATION

Build: `nix build` generates dark and light themes in share/pmd-base16/.
