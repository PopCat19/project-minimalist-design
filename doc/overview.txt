===== PROJECT MINIMALIST DESIGN - OVERVIEW =====

Project Minimalist Design (PMD) is a design system built on principles of
simplicity and practicality. The system emphasizes YAGNI (You Aren't Gonna
Need It) and DRY (Don't Repeat Yourself) principles.

Key Characteristics:
- Hand-picked -x foundation palette (hex values, canonical at each hue)
- Alpha compositing for UI surfaces (stack expressions: -x × opacity%)
- OKLCH used for perceptual hue rotation and gamut clamping only
- Hex as canonical color representation (matching Figma / palette-web)
- Framework-agnostic implementation
- Native Nix integration for theme generation

Foundation (-x Slots):
The color system is built on foundation slots defining specific lightness
levels. Each has a hand-picked canonical value at the reference hue:

  Slot    Role                  Dark l/c        Light l/c
  ─────   ────────────────────  ──────────────  ──────────────
  100x    max contrast          1.00 / 0.000    0.00 / 0.000
  88x     primary / headings    0.88 / 0.056    0.28 / 0.032
  80x     secondary / body      0.80 / 0.100    0.20 / 0.032
  72x     accent / subtext      0.72 / 0.122    0.32 / 0.052
  8x      base surface          0.20 / 0.032    0.88 / 0.056
  4x      deep background       0.16 / 0.022    0.92 / 0.044
  0x      canvas extreme        0.00 / 0.000    1.00 / 0.000

Derived Stack Expressions:
  80×8%  = composite(8x, 80x, 0.08)  interactable surface hint
  80×48% = composite(8x, 80x, 0.48)  muted text / inactive track
  88×24% = composite(8x, 88x, 0.24)  disabled foreground
  8×40%  = composite(0x, 8x, 0.40)   flyout / semi-base
  88×12  = 88x at hue column 12      priority alert

Reading Priority (highest → lowest):
  88x → 80x → 72x → 8x → 4x

Core Rules:
- No shadows, no gradients — depth via lightness only
- Borders only on priority alerts and control clusters
- All text: Fredoka, weights 500/600, sizes 10/12/16px
