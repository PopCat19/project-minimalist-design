===== FRAME CONVENTIONS =====

Border-Radius:
  16px (1rem) — Standard. Cards, standalone buttons, panels.
  8px        — Compact. Small cards, middle items in a group.
  40px       — Pill. Consolidated summary items.

Borders:
  Never used on standalone interactive elements. Only two cases:
    1. Priority alerts: 2px solid, alert-column 88x at 100%
    2. Control clusters: 2px solid, 80×8%

No Shadows, No Gradients:
  Depth is communicated through lightness tier alone.

Hover Feedback:
  Use a `--hout` custom property for hover outlines. Default
  value is 80x (body-text tier). Active/CTA elements (88x
  fill) override to 4x — the opposite tier always contrasts
  regardless of scheme. All hover rules reference `var(--hout)`,
  not per-component hardcodes.

Active Fill Text:
  Text on a filled active surface (88x toggle, filled track)
  uses the colour of the surface beneath the fill — not a
  computed contrast colour, not a fixed tier. If the surface
  immediately beneath is alpha-composited (80×8%, 80×48%),
  skip it and use the next opaque surface below. This keeps
  the text anchored to context rather than floating free.

Widget Surfaces:
  Built as 8x at 100% layered with 80×8% tint. This is the
  canonical widget background: composite(8x, 80x, 0.08).

Grouped Component Radii:
  Adjacent elements in a group use asymmetric radii to signal
  position without borders or dividers:
    - First:   16px followed by 8px on the shared side
    - Middle:  8px all around
    - Last:    8px followed by 16px on the shared side
    - Standalone: 16px all around
  This applies to button groups, bar widget groups, and welded
  panels alike.
