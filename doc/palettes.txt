===== PALETTE SYSTEM (HD SPECIFICATION) =====

PMD palettes are defined in YAML format mapping to base16 slots:

Base16 Slot | PMD Mapping           | Description
------------|-----------------------|--------------------------
base00      | 4x                    | True Background (Deepest Layer)
base01      | 8x                    | Base Container (Panels/Toolbars)
base02      | 8x+80x@16%            | Surface / Selection (Composited)
base03      | 80x @ 80%             | Muted / Comments (Baked)
base04      | 68x                   | Subtext / Secondary Metadata
base05      | 80x                   | Body Text (Main Reading)
base06      | 88x                   | Headers / Strong Text
base07      | 100x                  | Max Contrast Text/Symbol
base08      | 88x                   | Alert / Danger (Root Hue)
base09      | 68x +290°             | Constants (Hue Rotated)
base0A      | 80x                   | Warning / Classes (Root Hue)
base0B      | 68x                   | Strings (Root Hue)
base0C      | 80x +140°             | Support (Hue Rotated)
base0D      | 68x +30°              | Functions (Hue Rotated)
base0E      | 68x -30°              | Keywords (Hue Rotated)
base0F      | 68x @ 80%             | Deprecated (Meta / Baked)

Base16 YAML Structure:
The palette system exports to base16 YAML format with the following structure:
- scheme: "PMD [Hue] [Variant]" (e.g., "PMD 260 Dark")
- author: "Project Minimalist Design"
- variant: "dark" or "light"
- base00-base0F: Hex color values (6 characters, no # prefix)

Generation Process:
1. PMD scheme is defined in OKLCH color space
2. OKLCH values are converted to RGB via Newton-Raphson iteration
3. RGB values are converted to sRGB hex format
4. Output is written to base16 YAML files

Example Structure:
scheme: "PMD [Color Name]"
author: "Project Minimalist Design"
hue: [0-360]
aux_hue: [0-360]  # Default: (hue + 180) mod 360

Auxiliary Hue Calculation:
Default: aux_hue = (primary_hue + 180) mod 360
Example: hue 345° → aux_hue = (345 + 180) mod 360 = 165°
Example: hue 260° → aux_hue = (260 + 180) mod 360 = 80°

PMD HD Specification Logic:
- Tiered Visibility: Uses Lightness for global hierarchy
- Soft Content: Muted text (base03) is baked at 80% alpha
- Physical Metadata: Subtext (base04) is dropped to the 68x lightness tier
- Hue Rotation: Accent colors use hue offsets for distinction
- Functional Aux: Uses (Root + 180°) for complementary contrast

Hue Rotation System:
Accent colors use fixed hue offsets from the root hue:
- base09: +290° (Constants)
- base0C: +140° (Support)
- base0D: +30° (Functions)
- base0E: -30° (Keywords)

Color Definitions:
base00-base07: Grayscale and neutral tones
base08-base0F: Accent and semantic colors

Each palette maps to PMD's lightness-based system:
100x, 88x, 80x, 68x, 8x, 4x, 0x

Multiple opacity variants are specified for each base color.
Auxiliary hues provide complementary contrast (+180° typically).
